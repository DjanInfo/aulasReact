FER-016: Criando e Instanciando Classes

Implementar e utilizar classes em JavaScript

Boas Práticas

✅ Convenções de Nomenclatura

  • Classes: PascalCase (primeira letra maiúscula)
  • Métodos: camelCase (primeira letra minúscula)
  • Atributos: camelCase
  • Nomes descritivos: Evite abreviações
// ✅ BOM
class ContaBancaria {
    constructor(titular) {
        this.titular = titular;
    }
    
    depositar(valor) { }
}

// ❌ RUIM
class cb {
    constructor(t) {
        this.t = t;
    }
    
    dep(v) { }
}

✅ Organização do Código

  • Constructor sempre no início
  • Atributos inicializados no constructor
  • Métodos organizados logicamente
  • Um método, uma responsabilidade

✅ Validação no Constructor

class Produto {
    constructor(nome, preco) {
        // Validação
        if (!nome || nome.trim() === "") {
            throw new Error("Nome é obrigatório!");
        }
        if (preco < 0) {
            throw new Error("Preço não pode ser negativo!");
        }
        
        // Inicialização
        this.nome = nome;
        this.preco = preco;
    }
}

✅ Comentários e Documentação

/**
 * Classe que representa uma conta bancária
 */
class ContaBancaria {
    /**
     * Cria uma nova conta bancária
     * @param {string} titular - Nome do titular
     * @param {number} saldoInicial - Saldo inicial da conta
     */
    constructor(titular, saldoInicial = 0) {
        this.titular = titular;
        this.saldo = saldoInicial;

}

Evite

  • Classes muito grandes (muitas responsabilidades)
  • Métodos muito longos
  • Atributos públicos desnecessários
  • Lógica complexa no constructor
